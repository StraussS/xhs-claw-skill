# 复合运营工作流

## 什么时候读这个文件

当任务涉及：
- 竞品分析
- 热点追踪
- 研究后创作发布
- 批量互动策略
- 爆款内容观察

## 通用方法

把复合任务拆成：
1. 搜索
2. 筛选
3. 查看详情
4. 总结
5. 互动或创作发布

## 竞品分析

推荐流程：

```bash
python scripts/cli.py search-feeds --keyword "目标关键词" --sort-by 最多点赞
```

然后对 3-5 篇目标笔记取详情：

```bash
python scripts/cli.py get-feed-detail --feed-id FEED_ID --xsec-token XSEC_TOKEN
```

分析维度：
- 标题风格
- 封面特点
- 开头结构
- 标签策略
- 点赞/评论/收藏表现

## 热点追踪

同一关键词做两轮搜索：

```bash
python scripts/cli.py search-feeds --keyword "关键词" --sort-by 最新 --publish-time 一周内
python scripts/cli.py search-feeds --keyword "关键词" --sort-by 最多点赞
```

这样可以同时看：
- 最近在发什么
- 哪些内容最容易爆

## 研究后创作

流程建议：
1. 搜索主题关键词
2. 取 2-3 篇高互动样本详情
3. 提炼结构与表达方式
4. 生成新草稿，避免简单搬运
5. 用户确认后按分步发布执行

## 批量互动策略

只把它当成“半自动工作流”，不要做无脑高频互动。

建议：
- 先筛选相关度高的笔记
- 优先点赞 / 收藏
- 评论要有内容，不要模板化灌水
- 每次操作之间留出明显间隔

## 对用户汇报时的格式

优先给：
- 关键发现
- 代表样本
- 数据对比
- 下一步建议

如果需要横向对比，再整理成表格。
